L'Architecture d'abord (The "Why") :
 Avant de demander du code, demande-moi : "Quelle est la meilleure structure pour ce problème ?". Un expert se démarque par sa capacité à organiser son code (comme le MVC qu'on utilise).

L'Anticipation des Erreurs :
 Ne te contente pas d'un code qui marche. Demande : "Qu'est-ce qui pourrait faire planter ce code ?". C'est ce qui rend ton travail robuste pour du Remote.

La Refactorisation :
Une fois que ton code fonctionne, demande : "Comment rendre ce code plus élégant ou plus performant ?".
