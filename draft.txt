Je développe un jeu de Gomoku en Python appelé "Pointing Game". Voici le contexte complet :
Règles du jeu :

2 joueurs placent des pions (bleu = J1, rouge = J2) sur une grille n×n (min 5, max 30)
Gagner = aligner exactement 5 pions (pas 6+) horizontalement, verticalement ou en diagonale
Exception : si un joueur a 10 pions ou plus sur la grille, le seuil devient 4
Chaque victoire = 1 point, le jeu est infini jusqu'au bouton "Terminer"

Mécaniques spéciales :

Un canon à gauche suit la souris sur l'axe Y
Ctrl + 1~9 tire sur une colonne X (règle de 3 selon n) et détruit le premier pion adverse sur la ligne Y du canon
Le tir passe automatiquement au joueur suivant

Stack technique :

Python + Tkinter pour l'interface
MongoDB (localhost:27017, base pointing_game, collection parties)
Structure MVC :

controllers/game_logic.py — logique du jeu
views/game_view.py — interface de jeu
views/setup_view.py — formulaire de configuration
models/config.py — classe GameConfig (objet partagé entre tous les modules)
models/database.py — fonctions MongoDB
main.py — point d'entrée

Fonctionnalités déjà implémentées :

Placement de pions et détection de victoire (seuil dynamique 4 ou 5)
Score et label du joueur actuel
Canon avec tir Ctrl+1~9
Bouton Terminer → sauvegarde dans MongoDB
Historique des parties avec reprise (update si reprise, insert si nouvelle)
Bouton Historique sur la fenêtre de jeu ET sur la configuration
Lignes jade sauvegardées et restaurées à la reprise
